# WispTerm v1.13.0

> This release adds Copilot file-edit tools, OpenSSH config import, smarter
> terminal path actions, and WeChat-driven remote approval replies.

## Added

- **Copilot file tools.** Copilot can now use `read_file`, `write_file`, and
  `edit_file` against local terminal sessions and SSH-backed surfaces, with the
  same path-aware access gate used by shell tools.
- **Remote SSH file helpers.** Remote file reads and writes now go through
  out-of-band SCP helpers so Copilot can edit files on the active SSH
  connection without relying on terminal paste.
- **OpenSSH config import.** The command center can import `Host` entries from
  OpenSSH config files into WispTerm SSH profiles, including imported config
  files and multiple aliases.
- **Smarter terminal path actions.** Ctrl-click path previews now understand
  soft-wrapped paths, command-output path continuations, and bare filenames
  listed by `ls <dir>`.
- **Open in default app.** Ctrl-right-click can open a local file under the
  cursor in the OS default application.
- **R script previews.** Local `.r` and `.R` files are recognized as
  previewable text files.
- **WeChat approval replies.** When Copilot is waiting for approval, WispTerm
  can push an approval prompt to WeChat and route remote Y/N replies back into
  the active approval flow.
- **Copilot task context controls.** Copilot terminal context and task controls
  are exposed more directly in the UI.

## Changed

- **Path preview resolution.** Bare relative filenames are joined only when a
  nearby `ls` directory prefix is confidently inferred, and the same prefix is
  reused for local HTML preview clicks.
- **Copilot tool prompts.** The live Copilot system prompt now advertises file
  tools and carries enough terminal/SSH context for tool calls to target the
  right surface.
- **Session organization.** AI history sessions now categorize subagents, and
  the Copilot sidebar stays scoped to terminal tabs.
- **OpenSSH parser compatibility.** OpenSSH config parsing now keeps option
  scope inside host blocks and handles import edge cases more consistently.

## Fixed

- **Soft-wrapped path clicks.** Path tokens that cross terminal line wraps can
  now be clicked and previewed correctly.
- **Preview path continuation.** Path detection no longer loses continuations
  across line breaks, and prompt-budget handling for preview extraction is more
  stable.
- **Agent file edit edge cases.** `write_file` accepts empty content, `read_file`
  no longer adds a phantom trailing line, and identical edits produce a clearer
  result instead of a misleading diff.
- **SSH alias resolution.** File tools can resolve surface and SSH connection
  aliases more reliably before reading or editing remote files.
- **Mention marker trimming.** Mention markers are trimmed before preview path
  extraction so they do not pollute detected paths.

## Known limitations

- **Agent file-access enforcement is heuristic.** It gates ordinary tool and
  shell command paths and intentionally biases toward extra prompts for
  protected paths; it is not an OS sandbox.
- **macOS Intel artifact.** The automatic macOS release ships Apple Silicon
  (`aarch64`) only. The Intel `x86_64` DMG remains a manual build and is not
  included in this release.

# WispTerm v1.13.0（中文）

> 本版本新增 Copilot 文件编辑工具、OpenSSH config 导入、更智能的终端路径操作，以及
> 通过微信回复远程审批。

## 新增

- **Copilot 文件工具。** Copilot 现在可以在本地终端和 SSH surface 上使用
  `read_file`、`write_file`、`edit_file`，并复用 shell 工具同一套按路径判断的访问
  gate。
- **远程 SSH 文件辅助通道。** 远程文件读写现在通过独立 SCP helper 执行，让 Copilot
  可以编辑当前 SSH 连接上的文件，而不依赖终端粘贴。
- **OpenSSH config 导入。** 命令中心现在可以把 OpenSSH config 里的 `Host` 条目导入为
  WispTerm SSH profile，并支持被导入的 config 文件和多个 alias。
- **更智能的终端路径操作。** Ctrl-click 路径预览现在能理解软换行路径、命令输出里的
  路径续行，以及 `ls <dir>` 列出的裸文件名。
- **用默认应用打开。** Ctrl-right-click 可以用系统默认应用打开光标下的本地文件。
- **R 脚本预览。** 本地 `.r` 和 `.R` 文件会被识别为可预览的文本文件。
- **微信审批回复。** 当 Copilot 等待审批时，WispTerm 可以向微信推送审批提示，并把远程
  Y/N 回复路由回当前审批流程。
- **Copilot 任务上下文控制。** Copilot 的终端上下文和任务控制在 UI 中暴露得更直接。

## 调整

- **路径预览解析。** 只有在能可靠推断附近 `ls` 目录前缀时，裸相对文件名才会被拼接，
  同一前缀也会用于本地 HTML 预览点击。
- **Copilot 工具提示。** 实时 Copilot system prompt 现在会声明文件工具，并携带足够的
  终端/SSH 上下文，让工具调用能定位到正确 surface。
- **会话组织。** AI history sessions 现在会给 subagent 分类，Copilot sidebar 也会限制
  在终端标签页内显示。
- **OpenSSH parser 兼容性。** OpenSSH config 解析会把 option 作用域限制在 host block
  内，并更一致地处理 import 边界情况。

## 修复

- **软换行路径点击。** 跨终端换行的路径 token 现在可以正确点击和预览。
- **路径续行预览。** 路径检测不再丢失跨行续接内容，预览抽取的 prompt budget 处理也更
  稳定。
- **Agent 文件编辑边界情况。** `write_file` 支持空内容，`read_file` 不再添加虚假的
  末尾空行，内容相同的编辑会返回更清晰的结果，而不是误导性的 diff。
- **SSH alias 解析。** 文件工具在读取或编辑远程文件前，会更可靠地解析 surface 和 SSH
  connection alias。
- **Mention marker 修剪。** 路径预览提取前会先修剪 mention marker，避免污染识别出的
  路径。

## 已知限制

- **Agent 文件访问规则是启发式保护。** 它检查普通工具和 shell 命令路径，并会对受保护
  路径偏向额外确认；它不是 OS 级 sandbox。
- **macOS Intel 产物。** 自动 macOS 发布只包含 Apple Silicon（`aarch64`）版本。Intel
  `x86_64` DMG 仍然是手动构建，本次发布不包含它。
