# WispTerm v1.10.0

> This release restores true full permission for AI Agent workflows, fixes the
> Default AI settings row interaction, and adds an in-app What's New panel.

## Added

- **True full Agent permission.** Full permission mode now lets trusted Agent
  workflows continue without the extra dangerous-command confirmation that
  blocked file operations such as create, rename, delete, and format.
- **Guarded Agent permission level.** Users who still want the local sensitive
  file and dangerous-command safety checks can use the guarded permission level
  instead of full permission.
- **What's New after upgrade.** WispTerm now embeds the current release notes,
  records the last seen app version, and auto-shows a What's New panel once
  after upgrading.
- **What's New command.** The command center includes a What's New entry so the
  current release notes can be opened again at any time.
- **AI skill distillation.** AI skill workflows can now be distilled into a
  reusable skill artifact from inside WispTerm.

## Changed

- **Copilot and Sessions naming.** User-facing AI Agent and AI History surfaces
  now use the Copilot and Sessions names.
- **Default AI selection controls.** The Default AI settings row now cycles to
  the next profile with Enter, click, or Right arrow, and cycles to the previous
  profile with Left arrow.
- **Agent terminal targeting.** Agent terminal targeting is leaner and refuses
  bare REPL launches that would otherwise wait for interactive input.

## Fixed

- **Codex REPL submission.** Codex REPL input submission now sends the Enter key
  separately so commands are accepted reliably.
- **Agent REPL waits.** REPL wait detection now settles on the live surface
  snapshot instead of treating worker-empty snapshots as completion.
- **macOS login bash.** macOS login bash sessions now source `~/.bashrc` so
  interactive shell setup is applied.

# WispTerm v1.10.0（中文）

> 本版本恢复 AI Agent 工作流里的真正 full 权限，修复默认 AI 设置行交互，并新增应用内
> What's New 更新说明面板。

## 新增

- **真正的 Agent full 权限。** full 权限模式现在会让可信 Agent 工作流继续执行，不再
  因创建、重命名、删除、格式化等文件操作触发额外危险命令确认。
- **Agent guarded 权限级别。** 仍希望保留本机敏感文件与危险命令安全检查的用户，可以
  使用 guarded 权限级别，而不是 full 权限。
- **升级后的 What's New。** WispTerm 现在会把当前版本 release notes 嵌入应用，记录
  上次已查看版本，并在升级后自动显示一次 What's New 面板。
- **What's New 命令。** 命令中心新增 What's New 入口，可以随时重新打开当前版本说明。
- **AI skill distillation。** 现在可以在 WispTerm 内把 AI skill 工作流沉淀成可复用的
  skill artifact。

## 调整

- **Copilot 与 Sessions 命名。** 用户可见的 AI Agent 和 AI History 界面现在分别使用
  Copilot 与 Sessions 命名。
- **默认 AI 选择交互。** Default AI 设置行现在支持 Enter、点击或右方向键切换到下一个
  配置，左方向键切换到上一个配置。
- **Agent 终端目标选择。** Agent 终端目标选择更轻量，并会拒绝裸 REPL 启动，避免卡在
  交互式输入等待里。

## 修复

- **Codex REPL 提交。** Codex REPL 输入现在会把 Enter 单独发送，让命令提交更可靠。
- **Agent REPL 等待。** REPL 等待判断现在基于实时 surface snapshot，不再把 worker-empty
  snapshot 误判为完成。
- **macOS login bash。** macOS login bash session 现在会 source `~/.bashrc`，确保交互式
  shell 配置生效。
