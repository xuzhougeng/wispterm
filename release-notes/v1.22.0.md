# WispTerm v1.22.0

> This release adds iTerm2-style **tmux control-mode (`tmux -CC`) remote session
> persistence**, a downloadable **Windows diagnostic build** for troubleshooting,
> a **Search box in the Sessions panel**, and new split shortcuts. It also fixes a
> UI freeze when ctrl+clicking a remote file, an iGPU whole-pane garble, clipboard
> image paste on macOS/Linux, and several tmux and macOS issues.

## Added

- **tmux control-mode remote persistence (`tmux -CC`).** Attach to a remote
  `tmux` over SSH and have its windows map to tabs and its panes to splits, so a
  dropped connection can be reattached without losing your session. Includes
  per-pane working-directory/preview parity and live agent-state dots. (#216, #219)
- **Windows diagnostic build.** Every release now also ships a
  `wispterm-windows-debug-<version>.zip`: a console build that writes an on-disk
  log to `%APPDATA%\wispterm\wispterm-debug.log` and a crash report on failure, so
  hard-to-reproduce issues can be diagnosed from a user's machine. There's also a
  manual CI workflow to build it for any branch. See `docs/windows-debug-build.md`.
- **Sessions panel search.** The Sessions panel has a Search box — start typing to
  filter sessions, then arrow straight into the results.
- **Split shortcuts.** A "Split Down" shortcut plus Windows-Terminal-style `+` / `-`
  split chords. (#232)
- **Skill Center: WSL target on Windows.** WSL can be selected as a deploy/sync
  target for skills. (#225)
- **Clipboard image paste on macOS and Linux.** Pasting an image from the
  clipboard now works on macOS and Linux (previously Windows-only). (#228)

## Changed

- **Skill Center on the home page.** The Skill Center now lives on the home page,
  and the redundant "Update Skills" entry was removed. (#226)
- **`wisptermctl` in releases.** The standalone agent terminal-control CLI is now
  published with releases for all platforms in a single zip. (#230)
- **Ctrl+click preview keeps focus on the terminal.** Opening a preview by
  ctrl+clicking no longer moves keyboard focus off the terminal, and Ctrl+Shift+W
  now asks before closing a terminal pane (a preview pane still closes in one press).

## Fixed

- **Ctrl+click on a remote file could freeze the app.** A remote directory check
  ran synchronously on the UI thread with no wall-clock bound, so a hung SSH
  session froze the whole window. The probe is now bounded (≤5 s) with an ssh
  keepalive, turning a permanent freeze into a brief, logged failure. (#234)
- **WeChat connection crashes are now diagnosable.** The connect path logs to the
  on-disk diagnostic log and a crash handler captures the fault, so a crash when
  opening the WeChat connection can finally be traced. (#234)
- **iGPU whole-pane garble.** On weak integrated GPUs, the screen could fill with
  garbage after the font atlas grew (cached cell UVs weren't invalidated). (#224)
- **macOS file dialog.** The save/open dialog now honors filter types and appends
  the correct extension on save. (#229)
- **tmux stability.** Restored tmux controllers are freed at quit (no more
  allocator leaks), and HTML preview, per-pane status-dot position, and
  reattach-on-drop were fixed. (#227, #219)

## Known limitations

- **Linux is experimental.** The Linux AppImage is published for community
  testing and is not yet considered stable.
- **Windows diagnostic build verification.** The diagnostic build's runtime
  behavior (console log, crash file) is verified on real Windows; please attach
  `%APPDATA%\wispterm\wispterm-debug.log` and any `crash-*.txt` when reporting.

# WispTerm v1.22.0（中文）

> 本版本新增 iTerm2 风格的 **tmux 控制模式（`tmux -CC`）远程会话持久化**、可下载的
> **Windows 诊断版本**(用于排查问题)、**Sessions 面板搜索框**,以及新的分屏快捷键。
> 同时修复了 ctrl+click 远程文件时的界面卡死、核显整屏花屏、macOS/Linux 剪贴板图片
> 粘贴,以及若干 tmux 和 macOS 问题。

## 新增

- **tmux 控制模式远程持久化（`tmux -CC`）。** 通过 SSH 接入远程 `tmux`,其窗口映射为
  标签页、窗格映射为分屏;断线后可重新接入而不丢失会话。包含每个窗格的工作目录/预览
  对齐,以及实时的 agent 状态圆点。（#216、#219）
- **Windows 诊断版本。** 每次发布现在都会附带一个
  `wispterm-windows-debug-<版本>.zip`:一个带控制台的版本,会把日志写入
  `%APPDATA%\wispterm\wispterm-debug.log`,崩溃时还会写崩溃报告,以便从用户机器上排查
  难以复现的问题。还提供一个可对任意分支手动构建它的 CI 工作流。详见
  `docs/windows-debug-build.md`。
- **Sessions 面板搜索。** Sessions 面板新增搜索框——直接打字即可过滤会话,然后用方向键
  跳进结果。
- **分屏快捷键。** 新增「向下分屏」快捷键,以及 Windows Terminal 风格的 `+` / `-`
  分屏组合键。（#232）
- **Skill Center:Windows 上的 WSL 目标。** 可以把 WSL 选作 skill 的部署/同步目标。
  （#225）
- **macOS 和 Linux 剪贴板图片粘贴。** 从剪贴板粘贴图片现在在 macOS 和 Linux 上可用
  （此前仅 Windows）。（#228）

## 调整

- **Skill Center 移到主页。** Skill Center 现在位于主页,并移除了多余的「Update Skills」
  入口。（#226）
- **`wisptermctl` 随发布提供。** 独立的 agent 终端控制 CLI 现在随发布提供,全平台打包在
  一个 zip 里。（#230）
- **Ctrl+click 预览保持终端焦点。** 通过 ctrl+click 打开预览不再把键盘焦点移出终端;
  Ctrl+Shift+W 现在关闭终端窗格前会先确认(预览窗格仍一次按键即可关闭)。

## 修复

- **Ctrl+click 远程文件可能导致卡死。** 一个远程目录检查在 UI 线程上同步执行且没有墙钟
  上限,SSH 会话卡住时整个窗口就会冻结。现在该探测有上限(≤5 秒)并加了 ssh keepalive,
  把永久卡死变成短暂且有日志的失败。（#234）
- **微信连接崩溃现在可诊断。** 连接路径会写入磁盘诊断日志,崩溃处理器会捕获故障,因此
  打开微信连接时的崩溃终于可以被追踪。（#234）
- **核显整屏花屏。** 在较弱的核显上,字体图集扩容后屏幕可能整片花屏(缓存的 cell UV 未
  失效)。（#224）
- **macOS 文件对话框。** 保存/打开对话框现在会遵循过滤类型,并在保存时追加正确的扩展名。
  （#229）
- **tmux 稳定性。** 恢复的 tmux 控制器会在退出时释放(不再有分配器泄漏),并修复了 HTML
  预览、每个窗格的状态圆点位置以及断线重连。（#227、#219）

## 已知限制

- **Linux 仍为实验性质。** Linux AppImage 用于社区测试,尚不视为稳定版本。
- **Windows 诊断版本验证。** 诊断版本的运行时行为(控制台日志、崩溃文件)在真实 Windows
  上验证;反馈问题时请附上 `%APPDATA%\wispterm\wispterm-debug.log` 和任何 `crash-*.txt`。
