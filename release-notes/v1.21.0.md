# WispTerm v1.21.0

> This release adds installing skills directly from GitHub, arrow-key gallery
> navigation for preview panes, and a standalone agent terminal control CLI
> (`wisptermctl`). It also fixes a Windows WSL issue that could freeze SSH splits
> or pop a `wsl.exe` window, plus large-file preview streaming.

## Added

- **Install skills from GitHub.** In the Skill Center, press `g`, paste a GitHub
  URL (repo / tree / blob), pick the discovered `SKILL.md` directories from a
  checklist, and download them into your skills library — fetched via the GitHub
  Contents API.
- **Preview gallery navigation.** Preview panes now remember their source kind
  and let you move across sibling files (e.g. the other images/PDFs in the same
  folder) with the arrow keys.
- **Agent terminal control API (`wisptermctl`).** A standalone CLI client lets
  external agents (Claude Code / Codex / scripts) drive terminal surfaces over a
  local token-authenticated channel. It ships as a separate artifact
  (`zig build wisptermctl`), not bundled with the app. (issue #173)
- **In-panel SKILL.md preview.** The Skill Center can show a scrollable
  `SKILL.md` preview overlay, with native Windows scanning/preview that does not
  rely on a POSIX shell or WSL.

## Changed

- **Preview pane open/close UX.** Opening a preview focuses it, and preview panes
  gain an `×` close button.
- **Larger AI system-prompt budget.** The default system-prompt length budget was
  raised from 3200 to 4000.

## Fixed

- **Windows WSL: frozen SSH splits / stray `wsl.exe` window.** On machines
  without a usable WSL install (only the System32 stub, or WSL not updated),
  splitting an SSH tab or starting the app could spawn `wsl.exe` implicitly — a
  blocking "press a key to install WSL" window that had to be closed first. An
  SSH surface's remote `/home/...` cwd was being treated as a WSL guest path
  (`wsl.exe --list`), and WSL was offered with no detection. Cwd conversion now
  only runs for WSL surfaces, WSL availability is detected from the registry
  (no process, no window), and the WSL launcher entry is hidden when WSL is not
  installed. Machines *with* WSL also lose the first-SSH startup lag. (#223)
- **Large-file preview streaming.** Previewing large files over SSH is capped at
  16 MiB, and huge text/logs show a scrollable truncated head instead of failing.
- **Skill Center on native Windows.** Scanning, transfer, and `SKILL.md` preview
  work without a POSIX shell or WSL, and stray `g` keystrokes no longer leak into
  the URL field.
- **`wisptermctl` exit code.** Running it with no command now exits 2, not 0.

## Known limitations

- **Linux is experimental.** The Linux AppImage is published for community
  testing and is not yet considered stable.

# WispTerm v1.21.0（中文）

> 本版本新增「从 GitHub 安装 skill」、preview pane 的方向键画廊导航，以及独立的
> agent 终端控制 CLI（`wisptermctl`）。同时修复了 Windows 上可能冻结 SSH 分屏或
> 弹出 `wsl.exe` 窗口的 WSL 问题，以及大文件 preview 的流式处理。

## 新增

- **从 GitHub 安装 skill。** 在 Skill Center 按 `g`，粘贴 GitHub URL（repo / tree /
  blob），从清单中勾选发现的 `SKILL.md` 目录并下载到你的 skills 库——通过 GitHub
  Contents API 获取。
- **Preview 画廊导航。** Preview pane 现在会记住来源类型，并支持用方向键在同目录的
  兄弟文件之间切换（例如同一文件夹里的其它图片 / PDF）。
- **Agent 终端控制 API（`wisptermctl`）。** 一个独立 CLI 客户端，让外部 agent
  （Claude Code / Codex / 脚本）通过本地带 token 鉴权的通道驱动终端 surface。它作为
  独立产物发布（`zig build wisptermctl`），不打包进 app。（issue #173）
- **面板内 SKILL.md 预览。** Skill Center 可以显示可滚动的 `SKILL.md` 预览浮层，
  Windows 上原生扫描/预览，不依赖 POSIX shell 或 WSL。

## 调整

- **Preview pane 打开/关闭体验。** 打开 preview 会聚焦它，preview pane 新增 `×`
  关闭按钮。
- **更大的 AI system prompt 预算。** 默认 system prompt 长度预算从 3200 提升到 4000。

## 修复

- **Windows WSL：SSH 分屏冻结 / 莫名弹出 `wsl.exe` 窗口。** 在没有可用 WSL 的机器上
  （只有 System32 桩、或 WSL 未更新），分屏 SSH tab 或启动 app 可能会隐式 spawn
  `wsl.exe`——一个阻塞的「按任意键安装 WSL」窗口，必须先关掉。原因是 SSH surface 的
  远程 `/home/...` cwd 被当成 WSL 客户机路径（`wsl.exe --list`），且 WSL 选项没有任何
  检测。现在 cwd 转换只对 WSL surface 生效，WSL 是否可用改为读注册表检测（不启动进程、
  不弹窗），未安装 WSL 时隐藏启动器里的 WSL 项。装了 WSL 的机器也不再有首次 SSH 的
  启动延迟。（#223）
- **大文件 preview 流式处理。** 通过 SSH 预览大文件限制在 16 MiB，超大文本/日志会显示
  可滚动的截断头部而不是直接失败。
- **Windows 原生 Skill Center。** 扫描、传输和 `SKILL.md` 预览不依赖 POSIX shell 或
  WSL，多余的 `g` 按键也不会再漏进 URL 输入框。
- **`wisptermctl` 退出码。** 不带命令运行时现在退出码为 2，而不是 0。

## 已知限制

- **Linux 仍为实验性质。** Linux AppImage 用于社区测试，尚不视为稳定版本。
