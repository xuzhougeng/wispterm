# WispTerm v1.18.0

> This release makes SSH folder transfers first-class, improves Windows
> presentation latency, and hardens several SSH/Agent edge cases.

## Added

- **Recursive SSH folder transfer.** The remote File Explorer can download
  directories and upload folders through recursive SCP.
- **Folder upload shortcut.** `Shift+U` opens a folder picker for recursive
  uploads to the current remote directory.
- **Terminal-path folder downloads.** `Ctrl+Shift` click on SSH terminal output
  now supports downloading directories, including extensionless directory names
  such as `results` or `build`.
- **DXGI flip-model presentation on Windows.** WispTerm now prefers a pure
  DXGI/D3D11 flip-present path and falls back to GDI `SwapBuffers` when needed.

## Changed

- **Download cancellation cleanup is safer.** Cancelling a download removes only
  destinations created by the transfer, preserving pre-existing files and
  folders.
- **Folder-download progress is clearer.** Directory downloads show
  "calculating..." progress instead of trying to treat the destination folder as
  a regular file.
- **Port forwarding profile selection is more robust.** The Port Forwarding
  form now re-verifies focused profile state after relocking and avoids clipping
  labels in the form.

## Fixed

- **Dragged folder uploads.** Dragging a local folder into a remote File
  Explorer or SSH terminal now uses recursive SCP instead of failing with
  "not a regular file".
- **SSH directory download affordance.** `Ctrl+Shift` hover now underlines
  extensionless remote directory names so they can be clicked for download.
- **Unsafe remote names are rejected.** Remote entries containing separators or
  `..` are refused before local download paths are built.
- **SSH helper diagnostics stay visible.** SSH stderr draining and helper-output
  caps were hardened so failures stay diagnosable without unbounded capture.
- **Agent surface lifetime safety.** Worker-thread access to terminal surfaces
  is guarded against stale surface identity and ABA-style reuse.
- **Large attachment guards.** Kitty PNG dimensions, outbound Weixin
  attachments, and remote read capture are capped before they can consume
  excessive memory.
- **Render diagnostics config.** `wispterm-debug-render` is parsed correctly.

## Known limitations

- **No macOS x86_64 DMG for this release.** v1.18.0 intentionally skips the
  manual Intel macOS workflow and publishes the Apple Silicon macOS DMG only.
- **Linux is experimental.** The Linux AppImage is published for community
  testing and is not yet considered stable.

# WispTerm v1.18.0（中文）

> 本版本将 SSH 文件夹传输做成一等功能，改进 Windows 呈现路径，并强化若干
> SSH/Agent 边界情况。

## 新增

- **递归 SSH 文件夹传输。** 远程 File Explorer 现在可以通过递归 SCP 下载目录和上传文件夹。
- **文件夹上传快捷键。** `Shift+U` 会打开文件夹选择器，并递归上传到当前远程目录。
- **终端路径文件夹下载。** 在 SSH 终端输出上 `Ctrl+Shift` 点击现在支持下载目录，包括
  `results`、`build` 这类没有扩展名的目录名。
- **Windows DXGI flip-model 呈现。** WispTerm 现在优先使用纯 DXGI/D3D11
  flip-present 路径，必要时回退到 GDI `SwapBuffers`。

## 调整

- **下载取消清理更安全。** 取消下载时只删除本次传输创建的目标，保留原本已经存在的文件和目录。
- **文件夹下载进度更清晰。** 目录下载会显示 "calculating..."，不再把目标目录当作普通文件计算大小。
- **端口转发 profile 选择更稳健。** Port Forwarding 表单在重新加锁后会再次确认焦点 profile，
  并避免表单标签被裁剪。

## 修复

- **拖拽文件夹上传。** 将本地文件夹拖进远程 File Explorer 或 SSH 终端时，现在会使用递归 SCP，
  不再因为 "not a regular file" 失败。
- **SSH 目录下载提示。** `Ctrl+Shift` hover 现在会给无扩展名远程目录名显示下划线，使其可以点击下载。
- **拒绝不安全远程名称。** 远程条目名包含路径分隔符或 `..` 时，会在构造本地下载路径前被拒绝。
- **SSH helper 诊断保持可见。** 强化 stderr drain 和 helper 输出上限，避免无界捕获，同时保留真实错误信息。
- **Agent surface 生命周期安全。** worker 线程访问 terminal surface 时会防止陈旧 surface 身份和 ABA
  式复用问题。
- **大附件防护。** Kitty PNG 尺寸、Weixin 发出附件和远程读取捕获现在都会在耗费过多内存前被限制。
- **渲染诊断配置。** `wispterm-debug-render` 配置项现在能被正确解析。

## 已知限制

- **本次不发布 macOS x86_64 DMG。** v1.18.0 会跳过手动 Intel macOS workflow，只发布
  Apple Silicon macOS DMG。
- **Linux 仍为实验性质。** Linux AppImage 用于社区测试，尚不视为稳定版本。
