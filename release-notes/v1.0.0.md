# WispTerm v1.0.0

> 🎉 **The project is now WispTerm (formerly Phantty), and 1.0.0 marks a fresh start.**
> This is the "rename + first 1.0" release: it folds in the v0.35.1 macOS fixes and
> adds a built-in skill updater plus a new default tab layout. From here on we no
> longer keep backward compatibility with old Phantty versions.

## Renamed

- **Phantty → WispTerm.** The display name, the executable (`wispterm` / `wispterm.exe`),
  the config directory (`~/.config/wispterm`, `%APPDATA%\wispterm`,
  `~/Library/Application Support/wispterm`), the portable config file (`wispterm.conf`),
  the macOS bundle (`WispTerm.app` / `com.wispterm.terminal`), and the app icon are all updated.
- **No config migration.** The old `phantty` config directory is not read; WispTerm starts
  with fresh defaults. To keep your old settings, copy them into the new directory manually.

## Added

- **Built-in skill updater.** A new `/update-skills` slash command and the "Update Skills"
  command-palette action pull the latest skills from the GitHub repository into
  `<config>/plugins/skills` — so you get new skills without updating the app itself. The
  download runs in the background without blocking the UI and refreshes the active Agent
  session's skill list when it finishes.

## Changed

- **Default tab layout is now Shell + Agent.** A new window opens the local Shell first
  (focused), with the Agent session to its right (previously Agent + Shell with the Agent focused).

## Fixed

- Folds in the v0.35.1 macOS fixes: window maximize/restore, punctuation shortcuts
  (`Cmd+=` / `Cmd+-` / `Cmd+,` / `Cmd+[` / `Cmd+]` / `` Ctrl+` ``, etc.), and `Cmd`-click
  preview / `Cmd+Shift`-click download / `Cmd`-click open-URL matching the docs.

## Notes

- Windows behavior is unchanged.
- **macOS is still early — use with caution.** The AppKit / Metal / multi-window paths can
  still misbehave in uncovered scenarios; prefer Linux or Windows for rock-solid daily use.
- Release assets, in-app auto-update, and skill downloads now point at the
  `xuzhougeng/wispterm` repository.

# WispTerm v1.0.0（中文）

> 🎉 **项目正式更名为 WispTerm（原 Phantty），并以 1.0.0 作为全新起点。**
> 本次是「重命名 + 首个 1.0 版本」，整合了 v0.35.1 的 macOS 修复，并带来内置
> skill 更新与默认标签布局调整。从这里开始不再考虑对旧 Phantty 版本的向后兼容。

## 重命名

- **Phantty → WispTerm。** 显示名、可执行文件（`wispterm` / `wispterm.exe`）、
  配置目录（`~/.config/wispterm`、`%APPDATA%\wispterm`、
  `~/Library/Application Support/wispterm`）、便携配置文件（`wispterm.conf`）、
  macOS 应用包（`WispTerm.app` / `com.wispterm.terminal`）以及应用图标全部更新。
- **不做配置迁移。** 旧的 `phantty` 配置目录不会被读取，WispTerm 以全新默认配置
  启动；如需沿用旧配置，请手动把内容复制到新目录。

## 新增

- **内置 Skill 更新。** 新增 `/update-skills` 斜杠命令与命令面板「Update Skills」，
  可直接从 GitHub 仓库拉取最新 skills 安装到 `<config>/plugins/skills`——无需更新
  主程序即可获取最新 skills。下载在后台进行，不阻塞界面；完成后自动刷新当前
  Agent 会话的 skill 列表。

## 变更

- **默认标签布局改为 Shell + Agent。** 新窗口启动时默认先打开本地 Shell 并聚焦，
  Agent 会话位于其右侧（此前为 Agent + Shell 且默认聚焦 Agent）。

## 修复

- 整合 v0.35.1 的 macOS 修复：窗口最大化后无法还原、标点类快捷键
  （`Cmd+=` / `Cmd+-` / `Cmd+,` / `Cmd+[` / `Cmd+]` / `` Ctrl+` `` 等）失效、
  以及 `Cmd`-点击预览 / `Cmd+Shift`-点击下载 / `Cmd`-点击打开 URL 与文档不一致。

## 说明

- Windows 行为不变。
- **macOS 仍处于早期阶段，请谨慎使用**：AppKit / Metal / 多窗口路径仍可能在
  未覆盖的场景下出问题；需要日常稳定使用时建议优先 Linux 或 Windows 版本。
- 发布资产与应用内自更新、skill 下载现指向 `xuzhougeng/wispterm` 仓库。
