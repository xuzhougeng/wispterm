# WispTerm v1.29.0

> This release rebuilds Copilot history around a multi-file store and a richer
> command-center history view, adds `wisptermctl ui-state` for UI automation,
> and expands the real macOS end-to-end harness.

## Added

- **Multi-file Copilot history store (#295).** Agent history now stores an index
  plus per-session files, lazily loads full bodies on demand, uses
  filesystem-safe session filenames, and migrates legacy single-file history
  once.
- **Richer Copilot History view (#291, #298).** The command center now has one
  Copilot History entry for sidebar and AI Chat history, with relative-time date
  groups, source chips, live search, Tab source cycling, grouped rows, and
  filtered selection mapping.
- **`wisptermctl ui-state` (#300).** The control CLI can expose focused overlay
  semantics and UI state so automation can assert real app behavior instead of
  relying on screenshots alone.
- **macOS e2e harness P1 (#303).** macOS UI automation now covers real
  input-to-PTY flows, deterministic driver behavior, and richer failure dumps.
- **AI command reference docs (#296).** The AI/Copilot command surface is now
  documented in the user docs and wiki.

## Changed

- **Copilot history reads are lighter.** Global history is backed by the new
  MetaStore index and lazy body loading instead of eagerly reading every full
  conversation.
- **Agent integration setup is prompt-based (#292).** The integration installer
  flow now uses an explicit prompt and closes the integration prompt after copy.
- **Platform status docs were reconciled.** Roadmap, known issues, and platform
  documentation now reflect the current desktop and macOS testing state.

## Fixed

- **Copilot model-switch checkpoints (#299).** Switching models no longer loses
  the expected history checkpoint.
- **Copilot History filtered interactions (#298).** Mouse hit-testing, row
  activation, deletion, navigation, and scrollbar thumbs now operate against
  the filtered display rows.
- **Interrupted history migration (#295).** If migration is interrupted, the
  store completes the partial migration before rebuilding from session files,
  and corrupt session files are skipped safely.
- **File Explorer repainting (#289).** Sidebar scrolling and keyboard
  navigation mark the UI dirty so rows repaint immediately.
- **SSH HTML preview readiness (#290).** SSH-backed HTML preview readiness was
  hardened so preview startup does not race the server.
- **Agent status detection (#285).** Agent status detection now uses hook-based
  signals for more reliable state transitions.

## Known limitations

- **Linux is experimental.** The Linux AppImage is published for community
  testing and is not yet considered stable.
- **macOS remote access remains early.** The macOS relay transport is available
  but still needs more end-to-end field testing.

# WispTerm v1.29.0（中文）

> 本版本围绕多文件存储和更完整的命令中心历史视图重建 Copilot history，新增
> `wisptermctl ui-state` 供 UI 自动化使用，并扩展真实 macOS 端到端测试 harness。

## 新增

- **多文件 Copilot history 存储（#295）。** Agent history 现在使用 index 加每会话文件的
  存储结构，按需懒加载完整正文，使用文件系统安全的 session 文件名，并会从旧的单文件历史
  执行一次性迁移。
- **更完整的 Copilot History 视图（#291、#298）。** 命令中心现在只有一个 Copilot
  History 入口，统一 sidebar 与 AI Chat 历史，并支持相对时间日期分组、来源 chip、实时搜索、
  Tab 切换来源、分组行和过滤后 selection mapping。
- **`wisptermctl ui-state`（#300）。** 控制 CLI 现在可以暴露当前 focused overlay 语义和
  UI 状态，让自动化测试可以断言真实应用行为，而不只依赖截图。
- **macOS e2e harness P1（#303）。** macOS UI 自动化现在覆盖真实 input-to-PTY 流程、
  确定性的 driver 行为，以及更丰富的失败现场 dump。
- **AI command reference 文档（#296）。** AI/Copilot 命令面现在已写入用户文档和 wiki。

## 调整

- **Copilot history 读取更轻。** 全局 history 现在由新的 MetaStore index 和懒加载正文支撑，
  不再打开历史面板时急切读取所有完整会话。
- **Agent integration 设置改为 prompt-based（#292）。** 集成安装流程现在使用显式 prompt，
  复制后会关闭 integration prompt。
- **平台状态文档已对齐。** Roadmap、Known Issues 和平台文档已更新，以反映当前桌面端与
  macOS 测试状态。

## 修复

- **Copilot 模型切换 checkpoint（#299）。** 切换模型时不再丢失预期的 history checkpoint。
- **Copilot History 过滤后交互（#298）。** 鼠标命中测试、行激活、删除、导航和 scrollbar
  thumb 现在都基于过滤后的 display rows 工作。
- **中断后的 history migration（#295）。** 如果迁移被中断，store 会先完成部分迁移，再从
  session 文件重建；损坏的 session 文件也会被安全跳过。
- **File Explorer 重绘（#289）。** 侧边栏滚动和键盘导航现在会标记 UI dirty，行内容会立即重绘。
- **SSH HTML preview readiness（#290）。** SSH 后端 HTML preview readiness 已加固，避免
  preview 启动与 server 状态竞态。
- **Agent 状态检测（#285）。** Agent status detection 现在使用 hook-based 信号，使状态切换更可靠。

## 已知限制

- **Linux 仍为实验性质。** Linux AppImage 用于社区测试，尚不视为稳定版本。
- **macOS 远程访问仍为早期功能。** macOS 中继传输已经可用，但仍需要更多端到端实际验证。
