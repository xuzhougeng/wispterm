# WispTerm v1.12.0

> This release expands the embedded browser into a Jupyter/web workflow, adds
> macOS WebView support, and ships a bilingual GitHub Wiki user guide.

## Added

- **Jupyter panel workflow.** The command center can open Jupyter in the
  embedded browser, auto-detect tokenized Jupyter URLs from the focused
  terminal, and offer a picker when multiple notebook servers are visible.
- **Full-screen browser mode.** The embedded browser panel can now switch
  between side-panel and full-content display modes, with `Esc` closing the
  full-screen browser overlay.
- **Local HTML preview server.** Clicking local `.html` paths can open them in
  the embedded browser through a small local HTTP server instead of handing the
  file directly to the browser engine.
- **macOS embedded browser backend.** macOS builds now include a WKWebView-based
  embedded browser backend behind the same platform facade used by Windows.
- **Browser refresh control.** The browser toolbar now includes a refresh
  action, and the platform webview layer exposes reload support.
- **Bilingual GitHub Wiki staging.** The repository now contains an English and
  Chinese wiki source tree with pages for installation, configuration, panels,
  AI Copilot, Jupyter/browser workflows, remote access, and FAQ.

## Changed

- **Panel header hit testing.** Panel header button handling is generalized so
  browser close, mode toggle, and refresh controls share the same interaction
  path.
- **Jupyter URL capture.** Terminal snapshots preserve soft-wrapped URLs so
  long Jupyter links survive line wrapping before detection.
- **Docs coverage.** User-facing docs were synchronized with the wiki content
  for missing operations and browser/Jupyter workflows.

## Fixed

- **Blocking overlays over WebView.** Native webviews are hidden while blocking
  overlays are visible, preventing the browser from visually covering command
  center and modal UI.
- **Full-screen browser sizing.** The terminal grid keeps its side-panel size
  while the browser is full-screen, avoiding accidental reflow to a nearly
  one-column terminal.
- **AI chat slash suggestions.** Slash-command suggestion counts no longer use a
  stale value after the command list changes.

## Known limitations

- **macOS Intel artifact.** The automatic macOS release ships Apple Silicon
  (`aarch64`) only. The Intel `x86_64` DMG remains a manual build and is not
  included in this release.

# WispTerm v1.12.0（中文）

> 本版本把内置浏览器扩展为 Jupyter/Web 工作流，新增 macOS WebView 支持，并随仓库提供
> 中英文 GitHub Wiki 用户指南。

## 新增

- **Jupyter 面板工作流。** 命令中心现在可以在内置浏览器中打开 Jupyter，自动从当前
  终端识别带 token 的 Jupyter URL，并在发现多个 notebook server 时显示选择器。
- **浏览器全屏模式。** 内置浏览器面板现在可以在侧边面板和全内容区模式之间切换，全屏
  浏览器覆盖层可用 `Esc` 关闭。
- **本地 HTML 预览服务。** 点击本地 `.html` 路径时，可以通过轻量本地 HTTP server 在
  内置浏览器中打开，而不是直接把文件路径交给浏览器引擎。
- **macOS 内置浏览器后端。** macOS 构建现在通过同一套平台 facade 接入基于 WKWebView 的
  内置浏览器后端。
- **浏览器刷新按钮。** 浏览器工具栏新增刷新操作，平台 webview 层也暴露 reload 支持。
- **中英文 GitHub Wiki staging。** 仓库现在包含英文和中文 wiki 源文件，覆盖安装、配置、
  面板、AI Copilot、Jupyter/浏览器工作流、远程访问和 FAQ。

## 调整

- **面板标题栏命中测试。** 面板标题栏按钮处理已泛化，浏览器关闭、模式切换和刷新控件
  使用同一套交互路径。
- **Jupyter URL 捕获。** 终端快照会保留软换行 URL，让很长的 Jupyter 链接在自动检测前
  不会被换行拆断。
- **文档覆盖。** 用户文档已同步 wiki 中补齐的操作说明和浏览器/Jupyter 工作流。

## 修复

- **WebView 与阻塞覆盖层。** 阻塞式覆盖层显示时会隐藏原生 webview，避免浏览器盖住命令
  中心和 modal UI。
- **全屏浏览器尺寸。** 浏览器全屏时终端网格会保持侧边面板尺寸，避免终端被意外 reflow
  成接近一列宽。
- **AI chat slash 建议。** 命令列表变化后，slash-command 建议数量不再使用过期值。

## 已知限制

- **macOS Intel 产物。** 自动 macOS 发布只包含 Apple Silicon（`aarch64`）版本。Intel
  `x86_64` DMG 仍然是手动构建，本次发布不包含它。
