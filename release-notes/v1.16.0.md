# WispTerm v1.16.0

> This release adds Copilot long-term memory, makes configured shell launches
> more faithful on Windows, and fixes clickable-path and macOS keyboard edge
> cases.

## Added

- **Copilot long-term memory.** Copilot can now keep durable facts across
  sessions in global and project tiers under the WispTerm config directory,
  injecting a compact memory index into future requests and recalling full
  entries on demand.
- **Memory tools and slash commands.** The Agent can use `memory_save`,
  `memory_recall`, and `memory_delete`, while users can manage memory directly
  with `/remember`, `/memory`, and `/forget` plus Chinese aliases.
- **Memory configuration.** New `ai-memory-enabled` config controls the entire
  memory system; it defaults to enabled and is honored at startup and reload.

## Changed

- **Memory prompt framing.** The built-in Agent prompt now teaches memory usage,
  and injected memory is explicitly framed as background context to verify, not
  as instructions.
- **Project-scoped recall.** Project memories follow the active working
  directory, while global memories remain available across projects.

## Fixed

- **Configured Windows shell launches.** Local shell launchers and Agent+shell
  startup now respect the configured Windows shell command instead of falling
  back to the built-in default.
- **Clickable path parsing.** Path detection now handles leading colon
  separators and hard newlines correctly, reducing false or merged preview
  targets in terminal output.
- **macOS key code collisions.** Period and quote keys no longer collide with
  Delete or arrow key handling on macOS.
- **Agent access cleanup.** Agent file-access rules are released during shutdown.

## Known limitations

- **No macOS x86_64 DMG for this release.** v1.16.0 intentionally skips the
  manual Intel macOS workflow and publishes the Apple Silicon macOS DMG only.
- **Memory v1 is file-backed.** It does not include embeddings, semantic search,
  a GUI manager, or automatic end-of-conversation extraction yet.

# WispTerm v1.16.0（中文）

> 本版本新增 Copilot 长期记忆，改进 Windows 配置 shell 启动行为，并修复可点击路径与
> macOS 键盘边界问题。

## 新增

- **Copilot 长期记忆。** Copilot 现在可以把跨会话事实保存到 WispTerm 配置目录下的
  全局层和项目层，后续请求会注入紧凑的记忆索引，并可按需召回完整条目。
- **记忆工具与 slash 命令。** Agent 可使用 `memory_save`、`memory_recall` 和
  `memory_delete`；用户也可以通过 `/remember`、`/memory`、`/forget` 及中文别名直接
  管理记忆。
- **记忆配置开关。** 新增 `ai-memory-enabled` 配置控制整个记忆系统；默认开启，并在
  启动和配置重载时生效。

## 调整

- **记忆提示词边界。** 内置 Agent prompt 现在会说明如何使用记忆；注入的记忆会明确标记
  为需要核实的背景上下文，而不是指令。
- **项目级召回。** 项目记忆会跟随当前工作目录；全局记忆则跨项目可用。

## 修复

- **Windows 配置 shell 启动。** 本地 shell launcher 以及 Agent+shell 启动现在会遵守
  配置的 Windows shell 命令，不再回退到内置默认值。
- **可点击路径解析。** 路径检测现在能正确处理前导冒号分隔符和硬换行，减少终端输出中
  误选或合并预览目标的情况。
- **macOS 键码冲突。** macOS 上句号和引号键不再与 Delete 或方向键处理发生冲突。
- **Agent 访问规则清理。** Agent 文件访问规则现在会在 shutdown 时释放。

## 已知限制

- **本次不发布 macOS x86_64 DMG。** v1.16.0 会跳过手动 Intel macOS workflow，只发布
  Apple Silicon macOS DMG。
- **记忆 v1 是文件式实现。** 暂不包含 embeddings、语义搜索、GUI 管理器或对话结束后的
  自动抽取。
