# WispTerm v1.27.0

> This release fixes SSH-profile file transfers and remote HTML/Jupyter preview,
> adds binary tools to the Skill Center and a new `ask_user` agent tool, and
> lands several macOS keyboard and paste fixes.

## Added

- **Skill Center binary tools.** You can now register standalone executables as
  agent tools from the Skill Center — enable/disable each tool, and the Copilot
  dispatches them at runtime alongside built-in tools.
- **`ask_user` agent tool.** Copilot can pause mid-task and ask you a
  multiple-choice question, surfaced as a card (and pushed to WeChat), then
  continue once you answer.
- **Better `$` suggestion list.** The skill/tool suggestion palette in the chat
  composer now caps its height and scrolls, and shows the selected suggestion's
  full description in a side detail panel.

## Fixed

- **SSH-profile file transfers work again (#268).** `Ctrl+Shift+Click` download
  and the agent `copy_file` tool no longer fail with "connection is
  unavailable" in sessions opened from the built-in SSH profile launcher. The
  download gesture now routes by SSH launch kind, the tool surface carries the
  SSH connection metadata the agent needs, and failures report a clear status
  instead of doing nothing.
- **Remote HTML / Jupyter preview over SSH (#256, #257).** The browser preview
  tunnel cache now distinguishes connections that use a `ProxyJump`, so HTML and
  Jupyter previews open the correct tunnel instead of reusing the wrong one. SSH
  helper calls also gained keepalive (`ServerAliveInterval`/`CountMax`) so a
  stalled metadata probe fails fast instead of hanging.
- **Skill descriptions with YAML block scalars (#265).** Multi-line `description:`
  blocks now parse correctly and show up in the `$` palette.
- **macOS keyboard and paste.** Copy is bound to `Cmd+C` only (freeing terminal
  `Ctrl+V`), the native menu shortcuts now match the `Cmd` keybinds, pasting an
  image with `Cmd` works, and quitting no longer rings the system bell.
  (#267, #271, #272)
- **Skill Center tool import hardening.** Imports are confirmed and validated,
  staging is preserved through the confirm step, the probe exit code is widened
  to avoid a Windows crash (#274), status columns scale to the font so labels no
  longer clip (#275), and importing from an empty library is allowed.

## Known limitations

- **Linux is experimental.** The Linux AppImage is published for community
  testing and is not yet considered stable.
- **macOS remote access remains early.** The macOS relay transport is available
  but still needs more end-to-end field testing.

# WispTerm v1.27.0（中文）

> 本版本修复 SSH 配置文件会话下的文件传输与远程 HTML/Jupyter 预览，为 Skill Center
> 增加二进制工具、新增 `ask_user` agent 工具，并修复多个 macOS 键盘与粘贴问题。

## 新增

- **Skill Center 二进制工具。** 现在可以在 Skill Center 把独立可执行程序注册为 agent
  工具，逐个启用/停用，Copilot 会在运行时像调用内置工具一样调用它们。
- **`ask_user` agent 工具。** Copilot 可以在任务中途暂停，向你抛出一个多选问题（以卡片
  形式呈现并推送到微信），你作答后再继续。
- **更好的 `$` 建议列表。** 聊天输入框的技能/工具建议列表现在会限制高度并可滚动，并在
  侧边详情面板中显示当前选中项的完整描述。

## 修复

- **SSH 配置文件会话的文件传输恢复正常（#268）。** 从内置 SSH profile launcher 打开的
  会话中，`Ctrl+Shift+左键` 下载和 agent 的 `copy_file` 工具不再报 “connection is
  unavailable”。下载手势现在按 SSH 启动类型路由，工具快照会带上 agent 需要的 SSH 连接
  元数据，失败时也会给出明确状态而不是毫无反应。
- **SSH 下的远程 HTML / Jupyter 预览（#256、#257）。** browser 预览 tunnel 的缓存现在会
  区分使用了 `ProxyJump` 的连接，HTML 和 Jupyter 预览会打开正确的 tunnel，而不会复用错误
  的那个。SSH helper 调用也加入了 keepalive（`ServerAliveInterval`/`CountMax`），元数据
  探测卡住时会快速失败而不是一直挂着。
- **带 YAML block scalar 的技能描述（#265）。** 多行 `description:` 现在能正确解析并显示
  在 `$` 面板中。
- **macOS 键盘与粘贴。** 复制只绑定到 `Cmd+C`（释放终端 `Ctrl+V`），原生菜单快捷键与
  `Cmd` 键位对齐，`Cmd` 粘贴图片可用，退出时不再触发系统提示音。（#267、#271、#272）
- **Skill Center 工具导入加固。** 导入会经过确认与校验，确认步骤会保留暂存状态，探测退出
  码加宽以避免 Windows 崩溃（#274），状态列按字体缩放使标签不再被裁切（#275），并允许从空
  技能库导入。

## 已知限制

- **Linux 仍为实验性质。** Linux AppImage 用于社区测试，尚不视为稳定版本。
- **macOS 远程访问仍为早期功能。** macOS 中继传输已经可用，但仍需要更多端到端实际验证。
