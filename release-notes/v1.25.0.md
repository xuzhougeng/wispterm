# WispTerm v1.25.0

> This release fixes the visible `ssh.exe` window that could appear when opening
> SSH-hosted HTML pages or Jupyter URLs, strengthens the bundled diagnostics
> workflow, and adds smaller Copilot and WeChat workflow improvements.

## Added

- **Copilot composer history.** The Copilot input composer now keeps recent
  prompts available for reuse, making repeated debugging and reporting prompts
  easier to edit instead of retyping.
- **WeChat conversation commands.** The WeChat bridge now has pin-aware
  conversation listing and switching command support, including digest/status
  formatting for `/list` and `/switch`.
- **WeChat model profile commands.** WeChat-driven AI workflows can now use
  model profile slash commands, matching the broader direction of profile-based
  model switching.

## Changed

- **Diagnostics-first support flow.** The bundled `wispterm-diagnostics` skill
  now has stronger guidance for SSH image preview failures, HTML/browser preview
  failures, and SSH disconnects. The FAQ now directs Windows users to invoke
  `$wispterm-diagnostics` from WispTerm AI Chat or Copilot, and notes that the
  skill can be downloaded or updated through Skill Center.

## Fixed

- **SSH HTML/Jupyter no longer opens a visible helper window.** Ctrl-clicking
  SSH-hosted `.html` / `.htm` files or remote loopback URLs such as Jupyter Lab
  still uses an OpenSSH browser tunnel internally, but the tunnel helper now runs
  as a hidden background process instead of opening a visible `ssh.exe` console
  window on Windows. (#256, #257)

## Known limitations

- **Linux is experimental.** The Linux AppImage is published for community
  testing and is not yet considered stable.
- **macOS remote access remains early.** The macOS relay transport is available
  but still needs more end-to-end field testing.

# WispTerm v1.25.0（中文）

> 本版本修复在 SSH 中打开 HTML 页面或 Jupyter URL 时弹出可见 `ssh.exe` 窗口的问题，
> 强化内置诊断流程，并加入一些 Copilot 与 WeChat 工作流改进。

## 新增

- **Copilot 输入历史。** Copilot 输入框现在会保留最近提示词，便于反复调试和整理报告时
  直接修改复用，而不必重新输入。
- **WeChat 会话命令。** WeChat bridge 现在支持带置顶感知的会话列表与切换命令，并为
  `/list` 和 `/switch` 提供摘要与状态格式化。
- **WeChat 模型 profile 命令。** WeChat 驱动的 AI 工作流现在可以使用模型 profile
  slash commands，与 WispTerm 逐步采用 profile 化模型切换的方向保持一致。

## 调整

- **以诊断为入口的支持流程。** 内置 `wispterm-diagnostics` skill 现在强化了 SSH 图片预览、
  HTML/browser 预览以及 SSH 断连问题的采集指引。FAQ 现在引导 Windows 用户在 WispTerm
  AI Chat 或 Copilot 中调用 `$wispterm-diagnostics`，并说明可以通过 Skill Center 下载
  或更新该 skill。

## 修复

- **SSH HTML/Jupyter 不再弹出可见 helper 窗口。** 在 SSH 会话中 Ctrl-click 打开
  `.html` / `.htm` 文件或 Jupyter Lab 这类远端 loopback URL 时，WispTerm 仍会在内部使用
  OpenSSH browser tunnel，但 tunnel helper 现在会作为隐藏后台进程运行，不再在 Windows 上
  弹出可见的 `ssh.exe` 控制台窗口。（#256，#257）

## 已知限制

- **Linux 仍为实验性质。** Linux AppImage 用于社区测试，尚不视为稳定版本。
- **macOS 远程访问仍为早期功能。** macOS 中继传输已经可用，但仍需要更多端到端实际验证。
