# WispTerm v1.5.0

> WeChat direct grows up: the AI agent can now send files, images, and voice to
> your bound WeChat, agent finish/confirm notifications can be forwarded to your
> phone, and long-running answers are delivered reliably instead of silently
> dropped.

## Added

- **WeChat attachments from the agent.** The AI agent can send files, images, and
  voice (delivered as file attachments) to a bound WeChat user through the new
  `weixin_send_attachment` tool, and inbound WeChat voice messages are routed
  into AI Chat as transcribed text.
- **Agent notifications forwarded to WeChat.** New `weixin-notify-forward` opt-in
  (default off) forwards Claude Code / Codex finish and confirmation
  notifications to your bound WeChat account over the built-in iLink direct
  connection — no third-party relay and no token in the shell. Forwarding is
  skipped while you're actively viewing the pane, mirroring toast suppression.

## Fixed

- **Slow AI replies over WeChat.** Answers from tasks slower than ~3 minutes are
  now delivered within the ~30-minute WeChat `context_token` window instead of
  being abandoned. Dense early checkpoints are kept, a "still working" heartbeat
  is sent every 5 minutes, and a re-send notice is delivered if the window closes
  before the final answer is ready. Applied to both the embedded Zig poller and
  the remote bridge; `weixin-reply-timeout-ms` is now a no-op.

# WispTerm v1.5.0（中文）

> WeChat direct 进一步成熟：AI agent 现在可以向绑定的微信发送文件、图片和语音，
> agent 的完成/确认通知可转发到手机，长耗时回答也能可靠送达，不再被静默丢弃。

## 新增

- **Agent 微信附件。** AI agent 可通过新的 `weixin_send_attachment` 工具向绑定的
  微信用户发送文件、图片和语音（以文件附件形式送达）；收到的微信语音消息会转写为文本
  并作为输入路由进 AI Chat。
- **Agent 通知转发到微信。** 新增 `weixin-notify-forward` 开关（默认关闭），通过内置
  iLink 直连把 Claude Code / Codex 的完成与确认通知转发到绑定的微信账号——无需第三方
  中转，shell 中也不暴露 token。正在查看对应面板时会跳过转发，与桌面 toast 抑制逻辑
  一致。

## 修复

- **微信侧慢速 AI 回复。** 耗时超过约 3 分钟的任务回答，现在会在约 30 分钟的微信
  `context_token` 有效期内送达，而不再被放弃。保留了前期密集的进度检查点，每 5 分钟
  发送一次“仍在处理”心跳；若有效期在最终答案就绪前结束，会发送重发提示。该修复同时
  应用于内置 Zig poller 与 remote bridge；`weixin-reply-timeout-ms` 现已是 no-op。
