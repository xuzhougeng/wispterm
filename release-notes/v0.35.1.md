# Phantty v0.35.1

> ⚠️ **macOS 平台仍处于早期阶段，请谨慎使用。** 本次 fix release 修复了
> v0.35.0 在 macOS 上的若干窗口与快捷键问题，但 AppKit / Metal / 多窗口
> 路径仍可能在未覆盖的场景下出问题。如果你需要日常稳定使用，请暂时优先
> 使用 Linux 或 Windows 版本。

## Fixed

- **macOS：窗口最大化后无法还原。** 双击标题栏、`Option+Enter` 都能放大
  窗口，但再次操作无法还原。`showRestored` 只调用了 `makeKeyAndOrderFront`，
  并没有取消 AppKit 的 zoom 状态。改为在窗口已 zoom 时先调用一次 `zoom:`
  （AppKit 的切换语义）取消放大，双击标题栏与 `Option+Enter` 现在都能正常
  最大化 / 还原。
- **macOS：标点类快捷键全部失效。** `Cmd+=` / `Cmd+-` 调整字号、`Cmd+,`
  打开配置、`Cmd+[` / `Cmd+]` 切换面板、`Ctrl+` ` 呼出 Quake 等在 macOS 上
  均无响应。根因是按键翻译对 `=` `-` `[` `]` `,` `` ` `` 等标点键返回了原始
  字符码，而键位表与 Windows 后端都按 Windows 虚拟键码匹配。改为优先使用
  物理键的虚拟键码映射（文本输入走独立的字符事件路径，不受影响），这些
  快捷键现在全部生效；并额外接受 `Cmd+Shift+=`（带 Shift 的 `+`）放大字号。
- **macOS：`Cmd`-点击未生效（文档与实现不一致）。** 文档写明在 macOS 上用
  `Cmd`-点击预览文件、`Cmd+Shift`-点击下载 SSH 文件、`Cmd`-点击打开 URL，
  但实际处理只检查 `Ctrl`，且 macOS 的鼠标事件根本没有携带 `Cmd`（super）
  修饰键。现在已将 `Cmd` 修饰键贯穿鼠标事件链路，并在 macOS 上以 `Cmd`
  作为打开 / 预览 / 下载的主修饰键（`Ctrl`-点击保留给系统的次级点击）。
  顺带修复浏览器地址栏的 `Cmd+A` / `Cmd+V`、文件浏览器的 `Cmd+S` 下载，
  并避免 `Cmd`+字母误触发文件浏览器的单键操作（重命名 / 新建 / 删除）。

## Notes

- Windows 行为不变。
- macOS 构建以签名并公证的 `.dmg` 形式提供，分别面向 Apple Silicon
  （`aarch64`）与 Intel（`x86_64`）；见下方附件。
