实用 Agent 工作流

让 Agent 帮你配置终端

WispTerm 的 Agent 现在可以用内置工具保存 SSH server profile、连接已保存 profile、读取终端现场，并在首次启动时把 Agent 和 PowerShell 放在一起。

配置 SSH Profile Agent 工具
wispterm — Agent + PowerShell
agent
powershell
ssh: lab
你 帮我创建一个 SSH profile：lab，alice@192.0.2.10:2222，用我的 SSH 密钥。
WispTerm 工具请求
ssh_profile_save name=lab host=192.0.2.10 user=alice port=2222 auth_method=key identity_file=~/.ssh/id_ed25519
saved profile="lab" auth_method="key" identity_file_saved=true

ssh_profile_connect profile_name=lab
connected surface_id=03 tab=2

用一句话创建 SSH Profile

这个案例来自 issue #33：SSH 新建界面对新手不够直观，但 Agent 可以把自然语言请求转换成已保存的 profile。

怎么说

告诉 Agent profile 名称、host、user、port，以及认证方式：password、key 或 credentials。它会调用 ssh_profile_save，不再靠手写 profile 文件。

保存到哪里

profile 会进入 WispTerm 正常的 SSH server 列表。已有 profile 会按名称或 host 更新；新 profile 默认端口是 22。

凭证处理

password 会在审批文本和工具输出里隐藏原文。key 方式保存私钥路径；credentials 方式交给 OpenSSH config、默认密钥、ssh-agent 或平台凭证处理。

可以这样问：
帮我创建一个 SSH profile，名字叫 lab：
host 192.0.2.10，user alice，port 2222。
用 auth_method key，identity_file ~/.ssh/id_ed25519，然后连接它。

Agent 会用：
ssh_profile_save
ssh_profile_connect

启动时默认有 Agent 和 PowerShell

普通首次启动 WispTerm 时，第一个入口是 AI Agent，第二个入口是 PowerShell。session restore 和显式 cwd 启动会保留原来的行为。

已有 AI Profile

第一个 tab 是默认 AI profile，并强制以 Agent 模式打开。第二个 tab 是当前配置的 PowerShell 系列 shell。

还没有 AI Profile

WispTerm 会创建一个终端占位并在上面打开 AI profile 配置表单，同时仍然创建旁边的 PowerShell tab。

Restore 优先

当 restore-tabs-on-startup = true 且成功恢复上次 session 时，WispTerm 不会额外添加默认 Agent 或 PowerShell tab。
